### 哈希表

[**202.快乐数**](https://leetcode.cn/problems/happy-number/)

> 编写一个算法来判断一个数 `n` 是不是快乐数。
>
> **「快乐数」** 定义为：
>
> - 对于一个正整数，每一次将该数替换为它每个位置上的数字的平方和。
> - 然后重复这个过程直到这个数变为 1，也可能是 **无限循环** 但始终变不到 1。
> - 如果这个过程 **结果为** 1，那么这个数就是快乐数。
>
> 如果 `n` 是 *快乐数* 就返回 `true` ；不是，则返回 `false` 。
>
> ----

> **示例 1：**
>
> ```
> 输入：n = 19
> 输出：true
> 解释：
> 12 + 92 = 82
> 82 + 22 = 68
> 62 + 82 = 100
> 12 + 02 + 02 = 1
> ```
>
> **示例 2：**
>
> ```
> 输入：n = 2
> 输出：false
> ```
>
> ----

>**提示：**
>
>`1 <= n <= 231 - 1
>
>---



**思路:**

其实就是两步: 1.求和 2.**判断是否循环**(也是被无限循环吓住了)

**难点:**就是理解快乐数为什么不像 π 一样无限循环?

一个 10 位数最大约 99 亿，但它的各位平方和最大只有 92×10=81092×10=810。所以不管起点多大，经过一次计算，数字就会迅速掉到一个较小的范围内,越来越小

快乐数序列要么最终停在 1（1 本身也是循环），要么进入一个不含 1 的循环，**绝不会无限不循环地走下去**。



```go
func isHappy(n int) bool {
    m := make(map[int]bool)
    for n != 1 && !m[n] {
        n, m[n] = getSum(n), true
    }
    return n == 1
}

func getSum(n int) int {
    sum := 0
    for n > 0 {
        sum += (n % 10) * (n % 10)
        n = n / 10
    }
    return sum
}
```
